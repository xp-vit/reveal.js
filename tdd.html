TDD? What the heck is it?

Software development technique where you write your automated tests before(!) you write implementation

Software development technique where you write your automated tests together(!) with your implementation

Tools

Main tool: Test

Do I need tests?

Yes No

Check software quality

make sure that written code works as expected

Bonus: Find bugs before Implementation sent to QA (or even before bug introduced)

Protect Stability

make sure that code will continue working in future as you expected originally
Bonus: No need for full manual regression after adding new functionality Bonus: Easy refactoring Bonus: Protects you from unsafe changes Bonus: Allows you to be Agile - implement change requests faster

Unit vs. Integration Tests

Unit testing - is a software testing method by which individual units of source code.

Integration testing - is software testing method in which individual software modules are combined and tested as a group.

Mocking dependencies

- make a replica or imitation of behaviour, so that class under test has no knowledge whether the dependency is real or an imitation.

  • EasyMock
  • JMock
  • Mockito
  • PowerMock (!) warning

My Choice: Mockito

  • Better integration with Spring
  • Supports partial mocking
  • Better integration with JUnit
import static org.mockito.Mockito.*;
import static org.junit.Assert.*;
import java.util.Iterator;
import org.junit.Test;
....
@Test
public void shouldReturnHelloWorld(){
    //create mock
    Iterator iMock=mock(Iterator.class);
    when(iMock.next()).thenReturn("Hello").thenReturn("World");
    //invoke
    String result=iMock.next()+" "+iMock.next();
    //assert
    assertEquals("Hello World", result);
    verify(iMock, times(2)).next();
}

Easy Recipe to Cook Tests

  • Don’t be afraid and be Brave!!!
  • Prepare your test infrastructure
  • Test small (atomic) pieces of your functionality
  • Always Test: positive flow, boundary conditions, exceptions
  • Reproduce all manually found bugs with tests
  • Run tests on CI, collect and analyze metrics
  • Improve your environment constantly
  • Feel better and sleep well when tests are green

Huge Project and No Tests

  • Create Environment to run Integration Tests
  • Cover your major functionality with IT
  • Start refactoring to break your Project to smaller pieces and Unit test those pieces
  • Use IT from Point 2 to make sure Project is still Stable
